# 告别满屏广告的地图APP，我用这套开源路书工具重塑了旅行规划

旅行中最令人头疼的环节是什么？不是抢特价机票，也不是早起赶车，而是**“做攻略”**。

打开高德/百度地图看距离，切到备忘录记下时间，打开Excel算预算，最后还要去小红书翻看避坑指南……等所有行程规划完，满屏的广告和繁杂的App切换足以消磨掉大半的期待。市面上的地图软件本质上是为“导航”设计的，而非“多日多地点的行程规划”。

## 🗺️ 纯粹的规划体验：让地图回归工具本质

RoadbookMaker 的界面非常干净，所有的核心操作都围绕着“点”与“线”展开，符合直觉。

### 1. 随心所欲的“打点”与“连线”
不需要繁琐的搜索确认，你可以在地图上任意位置点击，添加标记点。给它换上合适的 Emoji（比如🏨代表酒店，☕️代表咖啡馆）。
支持全键盘快捷键：按 `A` 快速添加，按 `C` 呼出连线面板，或者直接在地图上**按住右键拖拽**，即可将两个地点连接起来。

### 2. 贴近真实出行的交通方式
连接线不仅仅是两点之间的一条直线。你可以为每一段行程设定交通方式（步行🚶、自驾🚗、高铁🚄、飞机✈️等）。系统会自动为你估算距离与时间，如果是高铁或飞机，面板里甚至贴心地内置了携程购票的快捷入口。

### 3. 日程与预算，一目了然
对于多日游，时间线往往容易混乱。RoadbookMaker 支持**按日期对节点进行分组**，你可以通过右上角的日历轻松筛选出“仅看第二天”的行程。同时，工具内置了**费用统计模块**，记录每天的开销，实时自动计算总预算，告别 Excel 记账。

## 🤖 AI 赋能：用自然语言“说”出你的行程

这是 RoadbookMaker 在近期更新中最令人兴奋的功能 —— **内置 AI 助手**。

过去做攻略，我们需要自己查资料、定点位。现在，你只需要唤出右下角的 AI 悬浮球。
*   **一句话生成行程**：直接对它说：*“帮我规划一个北京三日游，包含故宫、环球影城，第一天要在南锣鼓巷喝咖啡。”* AI 会理解你的需求，并**直接在地图上为你打好坐标点、连好路线、分配好日期**。
*   **快捷指令输出**：行程做完了想发朋友圈？直接在聊天框输入 `/xiaohongshu`，AI 会根据你的行程地图，瞬间为你生成一篇排版精美、带 emoji 的小红书文案。
*   **智能补全**：遇到模糊的地点，AI 会自动调用后台搜索接口确认精确坐标；甚至能为每一天的行程自动总结备忘录。

> *注：AI 助手功能为防止滥用，演示环境有一定的限制；自托管用户只需填入自己的 API Key（如 OpenAI 或国产大模型）即可无限制使用。*

## 📤 攻略的终点是分享：优雅的数据导出

一份好攻略，必须能方便地分享给旅伴。RoadbookMaker 提供了极其丰富的导出选项，满足各种场景：

1.  **独立交互式 HTML**：这是一个非常极客的功能。你可以将整个路书导出为一个 `.html` 文件。这个文件**内置了完整的地图数据和交互逻辑**。发给朋友，对方无论用手机还是电脑，无需安装任何 App，双击点开就能像在原始网页里一样查看地图、拖拽缩放。
2.  **优雅的长图分享 (PNG)**：针对微信群聊场景，一键生成包含“行程总览地图 + 每日地图切片 + 时间轴”的高清长图，交通方式、时间、耗时一目了然。
3.  **系统日历同步 (.ics)**：这是效率控的福音。导出 `.ics` 文件后，可以直接导入你的 Apple Calendar 或 Google 日历，行程将在你的手机上到点准时弹窗提醒。

## 🛡️ 把数据握在自己手里：离线优先与极简自部署

少数派的读者通常对隐私和工具的稳定性有更高的要求。RoadbookMaker 在架构设计上，将“控制权”完全交给了用户。

*   **PWA 与完全离线**：它支持 PWA 安装。你可以把它添加到手机桌面，像原生 App 一样全屏运行。更重要的是，它的核心数据保存在本地 `localStorage`，**即使在没有信号的高铁上或深山里，依然能打开查看和编辑你的路书**。
*   **免受 API Key 困扰**：市面上很多开源地图工具需要你自己去申请百度/高德的开发者 Key，不仅麻烦还容易暴露。RoadbookMaker 后端内置了强大的 API 代理，你直接打开就能用地图搜索，零配置。
*   **Serverless 一键白嫖部署**：如果你希望拥有云端同步功能（在线模式），也不需要去买昂贵的云服务器。项目支持 **Cloudflare Workers 一键部署**。点一下项目主页的部署按钮，利用 CF 免费额度的 KV 存储和 AI，你就能在 5 分钟内，零成本搭建一套属于你自己的全栈路书系统。同时，也提供了 Docker 镜像供 NAS 玩家本地部署。
